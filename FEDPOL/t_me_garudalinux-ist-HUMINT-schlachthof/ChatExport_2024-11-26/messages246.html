no, it does not save user settings but system root.

Snapshots of the /home subvolume (where user settings and files are stored) are disabled by default (you can enable timeline snapshots of home in btrfs-assistant).

Ubuntu, for example, has fixed releases like 22.04, 22.10 and so on. You will have to upgrade from 22.04 to 22.10, if you want to switch to the newest release.
This usually takes some time and a lot to download, but as long as you don't upgrade, you'll have a stable system. Think a bit about Win10 to Win11 upgrade. Drawback: you often have older versions of software.

Rolling distro means that (more or less) all packages are updated with every update. So you'll never have to worry about upgrading.
That means, that an issues that otherwise would have happened during upgrade can arise any time. The advantage is that you'll always have the latest version of software and can't fall out of support (which happens quite fast with Ubuntu, for example).
With Garuda, there also are snapshots you can just boot to get to an older version of the system and have it running again, but you'll probably have to investigate the issue (because usually it is something that will only affect a few users).
Also the update tool does a good job to solve known issues automatically.


I prefer rolling distros and tbh at first I was quite disappointed to learn that there are fixed releases in Linux (and most of the distros are fixed releases), always thought that all were rolling distros. 🤣
