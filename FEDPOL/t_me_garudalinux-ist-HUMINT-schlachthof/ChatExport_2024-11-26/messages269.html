Eg: we wanna do this with our fish shell's rc file.

Suppose we have a folder (I named mine .gitbackup) we move the file to this folder.

Then use ln -s ~/path/to/.gitbackup/fishconfigfileORdir ~/.config/fishconfigdir/ORfile

Use complete dir paths (i experienced trouble with this)

This means any changes made to the fish config files in the .gitbackup will now be reflected by the files in config dir's as well.

Create a git repo in this folder and push this repo to github.

Now you are on a new machine you git pull this repo and create these symlinks again removing the original file from it's place. This can be a tedious process so we can have pacman do this for us using pkgbuild file and also install all softwares we want (like installing fish and putting it's config files in the right place for us).
